Add a new workstream to `docs/WORKSTREAMS.md`.

Ask for the following details (or extract from context if already discussed):
1. **Workstream name** — descriptive title
2. **Goal** — what this workstream delivers
3. **Owner** — agent ID or human name
4. **Key tasks** — related task IDs from TASKS.md
5. **Notes** — any relevant context

Then:
1. Read `docs/WORKSTREAMS.md` to find the last workstream ID
2. Add a new row to the Active Workstreams table with the next ID (W02, W03, etc.)
3. Add a new detail section below with goal, key tasks, and notes
4. Set status to "Not Started" and today's date
5. Commit with message: `[docs] add: workstream W[NN] — [name]`
